Panoramica

Architetto del progetto. Ha definito la struttura signal-driven, scritto il core runtime (main.gd, panel_manager, game_manager), la gameplay logic (character controller, pet FSM, stress system, mess spawner) e l'intero layer UI del gioco. Ha coordinato il team come Project Supervisor — redigendo runbook, guide operative, report tecnici consolidati — e ha tenuto insieme il progetto nelle fasi di integrazione degli asset di Cristian e dello schema di Elia.

Contributi per area

Architettura

  • Progettazione signal-driven: 46 segnali SignalBus tipizzati, 14 domini (room, character, audio, UI, save, auth, cloud, stress/mood, mess, economy, profile HUD)
  • Definizione ordine autoload critico (10 singleton, ogni layer dipende solo dai precedenti)
  • Decisione architetturale offline-first: JSON + SQLite source-of-truth, Supabase opzionale
  • Pattern catalog-driven: contenuto in JSON, zero hardcoding
  • Definizione 9 pattern codificati (P-01..P-09) documentati in CONSOLIDATED_PROJECT_REPORT.md
